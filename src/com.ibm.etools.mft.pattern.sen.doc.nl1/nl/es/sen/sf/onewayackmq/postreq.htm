Tareas que deben completarse después de generar el patrón Fachada de servicio a WebSphere MQ: unidireccional con acuse de recibo

Tareas que debe completar después de haber generado el patrón.

  1. Revise el CCSID. Este patrón supone que el CCSID utilizado para las solicitudes SOAP es 1208. Si la aplicación de proveedor requiere mensajes con un CCSID diferente, este parámetro debe ser configurado en el subflujo RequestProcessing utilizando el editor de flujo de mensajes. Esta propiedad no se puede modificar con el editor de archivador de intermediario.
  2. Revise la configuración del nodo SOAP. En el caso de los WSDL complejos, puede ser necesario seleccionar el WSDL desde el editor de flujo de mensajes; este requisito se muestra en el archivo de resumen.
  3. Cree el archivo archivador de intermediario (BAR). Añada el flujo de mensajes generado al archivo BAR. En el editor del Archivador de intermediario, debe seleccionar la opción Compilar y poner en línea recurso antes de crear el archivo BAR. Los conjuntos de mensajes a los que el flujo hace referencia también deben añadirse.
  4. Configure el archivo archivador de intermediario. No es necesario efectuar ninguna configuración adicional del flujo de mensajes en el archivo archivador de intermediario antes de desplegar los flujos de mensajes para la instancia del patrón Fachada de servicio a WebSphere MQ: unidireccional con acuse de recibo, pero es posible que desee revisar algunas de las opciones predeterminadas y los cambios de configuración que deban aplicarse a su entorno. Por ejemplo, puede que desee revisar las siguientes propiedades:
    • LoggingOn. Si el registro está incluido, en el tiempo de despliegue este conmutador determina si es necesario el registro de solicitudes para el entorno actual.
    • ErrorLoggingOn. Si se incluyen mensajes de error, en tiempo de despliegue este cambio determina si es necesario el registro de errores para el entorno actual.
    Puede cambiar cualquiera de los parámetros de patrón que se listan en las tablas de parámetros como configurables, pero debe ir con cuidado porque el patrón podría asumir un comportamiento específico. Sin embargo, puede cambiar las propiedades determinadas por características operativas y que no afectan a la lógica del flujo como, por ejemplo, la configuración de seguridad o instancias adicionales.
  5. Cree colas. Cuando se aplica el patrón, se genera una lista de colas de WebSphere MQ, con nombres de cola y nombres de gestor de colas; estas colas de WebSphere MQ deben crearse si no existen ya. Todos los valores de mensaje que el patrón necesita se establecen explícitamente en los mensajes y no dependen de los valores de la cola.
    • En el archivo de resumen, vea la lista de colas que debe crear. El archivo de resumen se encuentra en la carpeta Configuración de patrón, en el proyecto de instancia del patrón Nombre_instancia.
    • Cree recursos de WebSphere MQ. Puede crear recursos de WebSphere MQ en la línea de mandatos utilizando programas y programas de utilidad, tales como runmqsc, o utilizando la interfaz gráfica de WebSphere MQ Explorer. Para obtener más información sobre la creación de recursos de WebSphere MQ, consulte WebSphere MQ Intercommunication, que está disponible en la página web de la biblioteca de WebSphere MQ.
  6. Despliegue el archivo BAR. El conjunto de mensajes que contiene el WSDL desplegable debe desplegarse, utilizando el mismo archivo BAR u otro, en todos los intermediarios en que se han desplegado los flujos.
