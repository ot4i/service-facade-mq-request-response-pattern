Service Facade to WebSphere MQ: request-response 型樣的參數

服務資訊
型樣參數 M C 預設值 說明
服務 WSDL 是 否 不適用 此型樣參數定義可部署的 WSDL 檔案，而此檔案定義分配管理系統向要求者呈現的服務介面。

此值必須包括從工作區訊息集專案開始的路徑名稱。

若要導覽至必要的 WSDL，請按一下瀏覽。
SOAP 要求的驗證 否 是 無 送入的服務要求會根據此參數的定義進行驗證。 有效值為：
  • 無
  • 內容和值
SOAP 回應的驗證 否 是 無 送出的 SOAP 回應會根據此參數的定義進行驗證。 有效值為：
  • 無
  • 內容和值
回應資訊
型樣參數 M C 預設值 說明
回應佇列 是 是 RESPONSE 此型樣參數識別提供者應用程式必須傳回回應的目標分配管理系統佇列。
儲存佇列 是 是 STORE 此型樣參數定義在等待提供者回應時用來儲存 MQMD 標頭的佇列。
儲存佇列訊息的期限 (以十分之一秒為單位) 是 是 不逾時 此型樣參數定義保留 HTTP 回覆 ID 的訊息的期限時間（以十分之一秒為單位）。

此值必須超出預期回應時間上限。

您可以輸入自己的值，或接受預設值不逾時。
提供者資訊
型樣參數 M C 預設值 說明
提供者佇列管理程式 否 是   此型樣參數識別提供者應用程式在其上接收分配管理系統訊息流程中的要求的佇列管理程式。

如果保留為空白，則預設值是分配管理系統佇列管理程式。
提供者要求佇列 是 是 PROVIDER 此型樣參數識別提供者應用程式在其上接收分配管理系統訊息流程中的要求的佇列。
記載
型樣參數 M C 預設值 說明
需要記載 否 否 已清除

此型樣參數決定型樣實例是否包括用於記載的程式碼。

如果清除需要記載，則不會包括「日誌」子流程。

如果選取需要記載，則「日誌」子流程會併入訊息流程中，而且必須設定日誌佇列管理程式及日誌佇列的值。
日誌佇列管理程式 否 是

此型樣參數定義佇列管理程式以進行記載。 如果使用分配管理系統佇列管理程式以進行記載，則可以將它保留為空白。

只有在選取需要記載時，才需要此參數。

日誌佇列 否 是 LOG

此型樣參數定義佇列以進行記載。

只有在選取需要記載時，才需要此參數。
錯誤處理
型樣參數 M C 預設值 說明
需要錯誤訊息 否 否 已選取

此型樣參數定義型樣應用程式是否建立訊息流程元素，以建立錯誤訊息。

如果選取需要錯誤訊息，則必須設定錯誤佇列管理程式及錯誤佇列的值。
錯誤佇列管理程式 否 是  

此型樣參數定義錯誤訊息的佇列管理程式。 如果使用分配管理系統佇列管理程式以進行記載，則可以將它保留為空白。
錯誤佇列 否 是 ERROR

此型樣參數定義錯誤訊息的佇列。

只有在選取需要錯誤訊息時，才需要此參數。

一般
型樣參數 M C 預設值 說明
分配管理系統綱目 否 否 mqsi 分配管理系統綱目可以用來建立型樣實例的名稱空間。

如果已設定，則會在具名分配管理系統綱目中建立所有訊息流程及 ESQL 檔案。 如果未設定，則會使用預設綱目。

流程字首 否 否   如果已設定，則會將此字首新增至所有訊息流程名稱的開頭。
流程字尾 否 否   如果已設定，則會將此字尾新增至所有訊息流程名稱的結尾。
佇列字首 否 否   如果已設定，則會將此字首新增至所有佇列名稱的開頭。

如果使用預設佇列名稱，則必須設定字首及（或）字尾，才能識別每一個型樣實例的佇列名稱。

佇列字尾 否 否   如果已設定，則會將此字尾新增至所有佇列名稱的結尾。
簡要說明 否 否   此說明會新增至所有產生的訊息流程的簡要說明內容。
詳細說明 否 否   此說明會新增至所有產生的訊息流程的詳細說明內容。

所有型樣參數會與其預設值一起列出。

標頭為 M 的直欄指出內容是否為必要的。 標頭為 C 的直欄指出內容是否為可配置的，這表示您可以在將訊息流程新增至分配管理系統保存檔 (BAR) 時變更值。

如果未指出任何預設值，則預設值是空字串。
