Log subflow for the Service Facade to WebSphere MQ: one-way with acknowledgment pattern

If the logging subflow is included in the request flow, the logging of requests is controlled by the user-defined property: LoggingOn.

Log messages are written as persistent WebSphere MQ messages to a WebSphere MQ queue. These log messages can then be processed separately in accordance with the standard practices within your organization. The message log contains the original message and the following information in an MQRFH2 header:

  • Time stamp
  • Broker name
  • Message flow name
